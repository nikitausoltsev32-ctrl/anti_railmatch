SYSTEM INSTRUCTION: RAILMATCH B2B PLATFORM DEVELOPER

1. КОНЦЕПЦИЯ ПРОДУКТА

RailMatch — это интеллектуальная B2B-экосистема для железнодорожной логистики. Платформа автоматизирует взаимодействие между Грузоотправителями (Shippers) и Владельцами вагонов (Owners).
Главная цель: Прямая связь, тендерная механика ставок и прозрачное управление парком без посредников.

2. ТЕХНИЧЕСКИЙ СТЕК И СТРУКТУРА (АКТУАЛЬНО)

Архитектура: React Component-Based Architecture (Vite).
Frontend: React 18, Tailwind CSS, Lucide-Icons.
Шрифт: 'Manrope', sans-serif (Google Fonts).
Бэкенд: Supabase (Auth, PostgreSQL DB, Real-time Channels).

СТРУКТУРА ПРОЕКТА:
- app.jsx: Главный контроллер, навигация, глобальное состояние, подписки Supabase.
- components/: Модульная структура UI (LandingScreen, AuthScreen, MyRequestsView, MyBidsView и т.д.).
- src/supabaseClient.js: Конфигурация клиента Supabase.

3. МОДЕЛЬ ДАННЫХ (SUPABASE)

- profiles: { id (uuid), company, inn, role (shipper/owner), phone, plan }
- requests: { id (uuid), shipperInn, stationFrom, stationTo, cargoType, wagonType, totalWagons, totalTons, fulfilledWagons, fulfilledTons, status (open/completed) }
- bids: { id (uuid), requestId, ownerId, ownerName, ownerPhone, price, wagons, tons, status (pending/accepted/rejected) }

4. ВИЗУАЛЬНЫЙ КОД (ПРЕМИАЛЬНЫЙ ФИНТЕХ)

Скругления: rounded-[2.5rem] для карточек, rounded-3xl для виджетов.
Эффекты: Glassmorphism (backdrop-blur), анимированные градиенты, тени shadow-xl/2xl.
Интерактивность: Анимации (float, glow), Hover-эффекты на кнопках (scale-95/active).
Темы: Полная поддержка Light/Dark тем через tailwind dark: классы.

5. ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

5.1. Landing Page
- Hero секция с анимированными плавающими элементами (float).
- Сетка Grid-background и мягкие свечения (glow).

5.2. Рабочий стол и Аналитика
- Ролевая аналитика: динамические боксы с микро-трендами.
- Цвета: Orange (проблемы/дефицит), Emerald (успех), Blue (основные метрики).
- AI Agent Bar: Умная строка ввода для поиска грузов и создания заявок.

5.3. Биржа и Сделки
- Реалтайм-обновление данных через Supabase Channels.
- Механика: Подача Bid -> Принятие Shipper-ом -> ОбновлениеfulfilledWagons -> Переход в Чат.
5.3. Биржа и Модель переговоров (Negotiation Flow)

Биржа: Список карточек с прогресс-барами.

AI Agent Bar: Интерактивная строка для текстовых запросов поиска/создания.
Отклик: Владелец нажимает "Ответить на заявку" -> Списание лимита -> Мгновенное создание чата и переход в раздел "Сообщения".
Лимит откликов: На одну заявку могут ответить максимум 15 пользователей. При достижении лимита кнопка отклика блокируется.
Сделка: Кнопка "Заключить сделку" доступна внутри чата. Если стороны не договариваются, чат закрывается, и слот (1 из 15) освобождается для общего пула.

5.4. Сообщения и Личный кабинет

Сообщения: Отдельная страница (вкладка в Header справа от "Мои заявки/ставки"). Список активных диалогов по конкретным грузам.

Кабинет: Профиль, Верификация (загрузка документов), Команда, Уведомления (Telegram).


6. ИНТЕРФЕЙС И ЯЗЫК
