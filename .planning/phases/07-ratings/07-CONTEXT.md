**Gathered:** 2026-03-30

<domain>
## Phase Boundary

После раскрытия контактов (`contacts_revealed`) обе стороны могут подтвердить завершение сделки и оставить оценку партнёру (1–5 звёзд + комментарий). Рейтинг отображается в профиле и доступен для просмотра из чата.

</domain>

<decisions>
## Implementation Decisions

### Триггер оценки
- Кнопка **«Подтвердить завершение сделки»** появляется в чате после статуса `contacts_revealed`
- После нажатия открывается модал **«Оцените партнёра»**
- Каждая сторона подтверждает и оценивает независимо — не нужно ждать партнёра
- Одна оценка на сделку (bid) от каждого участника — повторно оценить нельзя
- Дедлайна нет — можно оценить в любое время после раскрытия контактов

### Содержание оценки
- **Звёзды 1–5** (обязательно)
- **Предустановленные теги** (выбрать один или несколько):
  - Положительные: «Надёжный», «Оперативный», «Честный», «Рекомендую»
  - Отрицательные: «Медлительный», «Не отвечал», «Проблемы с оплатой», «Не рекомендую»
- **Свободный текст** (опционально, короткий комментарий)

### Отображение рейтинга
- В шапке чата рядом с именем партнёра — кнопка **«Отзывы»** (иконка + средняя оценка, если есть)
- По нажатию — модал с:
  - Средней оценкой + количество отзывов
  - Счётчики: сколько положительных (≥4 звёзд) / отрицательных (≤2 звёзд)
  - Список отзывов: звёзды + теги + текст + дата (без имени автора — анонимно)
- В RequestCard и профиле — средняя оценка со звёздочкой (если ≥1 отзыва)

### Анонимность
- Отзывы анонимны — автор не показывается
- Пользователь видит свой собственный отзыв как «Ваш отзыв»

### Claude's Discretion
- Дизайн модала оценки и модала просмотра отзывов
- Цветовая кодировка звёзд (золотые/серые)
- Порядок сортировки отзывов (по дате, новые сверху)
- Как отображать частичный рейтинг (например 4.3 ★)

</decisions>

- `app.jsx:991` — `handlePayCommission` — точка где устанавливается `contacts_revealed = true` — рядом добавить логику для кнопки завершения

- `RequestCard.jsx` — отображает имя создателя + verification badge — сюда добавляется средняя оценка

### Established Patterns
- Статусы сделки хранятся в таблице `bids` (колонка `status`)
- Supabase Realtime — подписки на изменения уже настроены для bids/messages
- Edge Functions не нужны — рейтинг пишется напрямую через Supabase client с RLS

### Integration Points
- Новая таблица `reviews`: from_user_id, to_user_id, bid_id, rating, tags[], comment, created_at
- Денормализованные поля на `profiles`: average_rating NUMERIC(2,1), review_count INTEGER
- Кнопка «Подтвердить завершение» → новое поле на `bids`: `completed_by_shipper`, `completed_by_owner` (BOOLEAN)
- Полное завершение сделки когда оба подтвердили → `status = 'completed'`

<specifics>
## Specific Ideas

- Кнопка «Подтвердить завершение сделки» — в области действий чата, после раскрытия контактов, вместо или рядом с текущими кнопками оплаты
- Модал оценки: сначала звёзды → потом теги (меняются в зависимости от выбранных звёзд: 1-2 звезды показывают негативные теги, 4-5 — позитивные, 3 — все) → потом текст
- Кнопка «Отзывы» в шапке чата: небольшая, не перегружать UI — иконка Star + число или «★ 4.8»

</specifics>
