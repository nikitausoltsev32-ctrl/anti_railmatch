# Requirements: RailMatch Anti

**Defined:** 2026-03-21
**Core Value:** Защищённый чат с антиутечкой, гарантирующий монетизацию через платформу

## v1 Requirements (Production Launch)

### Стабилизация

- [x] **STAB-01**: Пользователь может зарегистрироваться и получить правильную роль (shipper/owner)
- [x] **STAB-02**: Все SQL миграции применены в правильном порядке без конфликтов
- [x] **STAB-03**: Realtime подписки работают для requests, bids, messages
- [x] **STAB-04**: Полный pipeline сделки работает end-to-end (заявка → ставка → чат → комиссия → контакты)
- [x] **STAB-05**: RLS policies корректны — пользователь видит только свои данные там где нужно
- [x] **STAB-06**: Быстрые действия в UserDashboard ведут на реальные разделы

### Уведомления (Telegram)

- [x] **NOTIF-01**: Грузоотправитель получает Telegram-уведомление когда владелец откликнулся на заявку
- [x] **NOTIF-02**: Пользователь получает уведомление о новом сообщении в чате сделки
- [x] **NOTIF-03**: Владелец получает уведомление когда ставка принята или отклонена
- [x] **NOTIF-04**: Обе стороны получают уведомление при раскрытии контактов

### Уведомления (Email)

- [ ] **EMAIL-01**: Resend API key подключён в Supabase secrets
- [ ] **EMAIL-02**: Email дублирует ключевые Telegram-уведомления (NOTIF-01..04)

### Верификация компаний

- [ ] **VERIF-01**: Admin может подтвердить или отклонить загруженные документы компании
- [ ] **VERIF-02**: После подтверждения verification_status обновляется в profiles
- [ ] **VERIF-03**: Верифицированные пользователи получают расширенные лимиты

### Аналитика

- [ ] **ANAL-01**: Загрузка парка считается из реальных данных bids/wagons, не hardcoded
- [ ] **ANAL-02**: Карта вагонов отображает реальные данные из таблицы wagons
- [ ] **ANAL-03**: KPI дашборд использует реальные агрегаты из БД

## v2 Requirements

### Платежи

- **PAY-01**: Интеграция Tinkoff / ЮKassa для оплаты комиссии платформы
- **PAY-02**: Автоматическое раскрытие контактов после подтверждения оплаты

### Рейтинг пользователей

- [ ] **RATING-01**: Кнопка «Подтвердить завершение сделки» появляется в чате после contacts_revealed; каждая сторона подтверждает независимо
- [ ] **RATING-02**: После нажатия открывается модал оценки: звёзды 1–5, теги (фильтруются по звёздам), необязательный текст
- [ ] **RATING-03**: Оценка сохраняется в таблице reviews; повторная оценка по той же сделке заблокирована
- [ ] **RATING-04**: После отправки оценки кнопка заменяется надписью «Вы оценили партнёра»
- [ ] **RATING-05**: Кнопка «Отзывы» в шапке чата после contacts_revealed; по нажатию — модал с анонимными отзывами (средняя + счётчики + список)
- [ ] **RATING-06**: Средняя оценка партнёра отображается в RequestCard и на кнопке «Отзывы» в чате
