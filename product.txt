Техническое задание: Платформа RailMatch (MVP)1. Общее описаниеRailMatch — это интеллектуальная B2B-экосистема для железнодорожной логистики, обеспечивающая прямую связь между владельцами грузов (Грузоотправителями) и владельцами подвижного состава (Владельцами вагонов).Цель: Устранение посредников, снижение стоимости перевозки за счет тендерной механики и автоматизация поиска с помощью AI.2. Ролевая модельПлатформа поддерживает три режима доступа:Грузоотправитель (Shipper): Публикует заявки на перевозку, принимает ставки от владельцев, выбирает исполнителя.Владелец вагонов (Owner): Ищет подходящие грузы, делает ставки на перевозку (полный или частичный выкуп объема).Демо-режим: Гостевой доступ для просмотра актуальных заявок без возможности совершать транзакции.3. Функциональные модули3.1. Landing Page (Входная группа)Hero-блок: УТП, описание преимуществ для обеих ролей.Конверсионные действия: Кнопки «Регистрация» (с выбором роли) и «Смотреть демо».Дизайн: Современный финтех-стиль, поддержка светлой и темной темы.3.2. Биржа (Каталог заявок)Real-time обновление: Данные синхронизируются с базой данных Firebase Firestore в реальном времени.Карточки грузов: Отображение маршрута (Откуда -> Куда), типа груза, типа вагона и прогресс-бара погрузки.AI Agent Bar: Умная строка поиска/создания для обработки текстовых запросов (концепт).Дашборд аналитики: Виджеты активности (ставки, выручка, конверсия).3.3. Система ставок (Bidding)Модальное окно: Ввод цены за 1 вагон и количества предлагаемых вагонов.Лимиты: Система списания «бесплатных ставок» (SaaS-монетизация).Статусы сделки: pending (ожидает), accepted (принята), completed (завершена).3.4. Личный кабинетВерификация: Блок загрузки уставных документов (ЕГРЮЛ, приказы).Управление командой: Возможность добавления нескольких логистов на один ИНН.Telegram-интеграция: Привязка бота по коду для получения уведомлений о сделках.Биллинг: История транзакций и покупка пакетов ставок.3.5. Мессенджер (Чат)Активируется автоматически после принятия ставки отправителем.Отображает прямые контакты контрагента (телефон, проверенный ИНН).4. Технический стекFrontend: React 18, Tailwind CSS (Styling).Icons: Lucide-React.Animations: CSS Animate / Framer Motion.Backend/DB: Firebase Firestore (Real-time).Auth: Firebase Auth (Anonymous & Custom Tokens).5. Структура Базы Данных (Firestore)Коллекция requests (Заявки)stationFrom, stationTo: Станции маршрута.cargoType, wagonType: Тип груза и вагона.totalWagons: Общий объем заявки.fulfilledWagons: Текущий объем выкупа.shipperInn: Идентификатор компании-отправителя.Коллекция bids (Ставки)requestId: Ссылка на заявку.ownerId: ID владельца вагонов.price: Предложенная цена.wagons: Кол-во вагонов в предложении.status: Состояние предложения.6. Дизайн-системаШрифт: Manrope (Геометрический гротеск).Цвета: * Primary: Blue-600 (#2563EB).Dark BG: Indigo-950 (#0B1120).Success: Emerald-500.Warning: Orange-500.Эффекты: Glassmorphism (Header), Soft Shadows, Smooth Transitions.
